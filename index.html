SAP SD Concept: Pre-Sales Documents
Inquiry (VA11)

Customer request for price/availability. Non-binding. Creates reference for quotation.

Binding price offer valid for a period. Can be converted to sales order directly.

Sales Order (VA01)

Legal commitment. Triggers availability check, credit check, and delivery creation.

Pricing Concept Explained
Condition Type

Defines how a price element behaves — is it a price, discount, surcharge, or tax? Each has a calculation type and sign (+/-).

Access Sequence

The search strategy: SAP looks up condition records from most-specific to least-specific (e.g. Customer+Material → Customer → Material).

Pricing Procedure

The sequence of condition types applied in order. Determined by Sales Org + Customer Pricing Procedure + Doc. Pricing Procedure.

Credit Check Types in SAP SD
Static Check

Compares all open items (orders + deliveries + invoices) against credit limit

Dynamic Check

Adds expected future orders within a horizon period to the exposure calculation

Oldest Open Item

Blocks order if customer has overdue items older than a specified number of days

Maximum Dunning

Blocks if customer dunning level exceeds a configured maximum level
